# Proyecto Final – Análisis de Ventas del Sistema

## Contexto

La empresa desea analizar la información almacenada en su sistema de ventas para comprender mejor el comportamiento de sus clientes, productos y vendedores.

Para ello, el equipo de análisis de datos debe construir una serie de consultas SQL que permitan generar **información estratégica para la toma de decisiones**.

El objetivo de este proyecto es aplicar los conocimientos adquiridos durante el laboratorio SQL para obtener información relevante desde la base de datos.

---

# Requisitos del proyecto

Para completar este proyecto, el estudiante deberá:

- utilizar múltiples tablas relacionadas
- aplicar filtros y condiciones
- utilizar agregaciones
- construir consultas con JOIN
- utilizar funciones analíticas
- aplicar subconsultas o CTE cuando sea necesario

---

# Parte 1 – Análisis de clientes

1. Identificar los 10 clientes que más dinero han gastado en el sistema.
2. Identificar los clientes que han realizado más compras.
3. Mostrar el total de compras realizadas por cada cliente.
4. Identificar los clientes que nunca han realizado una compra.
5. Mostrar el promedio de compra por cliente.

---

# Parte 2 – Análisis de productos

6. Identificar los 10 productos más vendidos.
7. Identificar los productos que nunca han sido vendidos.
8. Mostrar la cantidad total vendida de cada producto.
9. Mostrar el ingreso total generado por cada producto.
10. Identificar los productos con precio superior al promedio.

---

# Parte 3 – Análisis de ventas

11. Mostrar el total de ventas registradas en el sistema.
12. Identificar los días con mayor volumen de ventas.
13. Calcular el promedio de ventas por día.
14. Identificar las ventas cuyo monto supera el promedio general.
15. Mostrar el ranking de las ventas más altas.

---

# Parte 4 – Análisis de usuarios

16. Identificar los usuarios que han registrado más ventas.
17. Calcular el total vendido por cada usuario.
18. Mostrar el promedio de ventas por usuario.
19. Identificar los usuarios que no han registrado ventas.
20. Mostrar el ranking de usuarios según total vendido.

---

# Parte 5 – Análisis de inventario

21. Mostrar el stock disponible de cada producto por sucursal.
22. Identificar los productos con menor stock.
23. Identificar los productos con mayor stock.
24. Mostrar el total de productos disponibles en cada sucursal.
25. Identificar las sucursales con mayor cantidad de productos.

---

# Parte 6 – Análisis de categorías

26. Identificar las categorías con mayor número de productos.
27. Calcular el precio promedio de los productos por categoría.
28. Identificar las categorías con mayor volumen de ventas.
29. Identificar las categorías con menor número de productos vendidos.
30. Mostrar el ranking de categorías según ingresos generados.

---

# Parte 7 – Análisis avanzado

31. Identificar los productos cuyo precio está sobre el promedio de su categoría.
32. Mostrar el total vendido por mes.
33. Mostrar el total vendido por cliente y categoría.
34. Identificar los clientes que compran productos de más categorías distintas.
35. Mostrar el ranking de productos más vendidos dentro de cada categoría.

---

# Desafío final

Construir una consulta que genere un **reporte completo de ventas**, incluyendo:

- nombre del cliente
- producto vendido
- categoría del producto
- cantidad vendida
- precio
- total de la venta
- fecha de la venta
- usuario que registró la venta

El reporte debe permitir analizar **qué productos compran los clientes y quién registra las ventas**.

---

# Objetivo pedagógico

Al finalizar este proyecto el estudiante deberá ser capaz de:

- analizar información desde múltiples tablas
- generar reportes complejos
- comprender relaciones entre entidades
- utilizar funciones avanzadas de SQL
- transformar datos en información útil para el negocio
